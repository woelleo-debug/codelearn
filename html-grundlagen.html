HTML Grundlagen

HTML (HyperText Markup Language) ist die Grundlage jeder Webseite. In diesem Modul lernst du alles von der Dokumentstruktur bis zu semantischen Elementen – mit echten Code-Beispielen und klaren Erklärungen.

📖 Was ist HTML?

HTML ist die Auszeichnungssprache des Webs. Sie beschreibt die Struktur und den Inhalt einer Webseite – nicht das Aussehen (das ist Aufgabe von CSS) und nicht das Verhalten (das ist JavaScript).

HTML besteht aus Tags (Elementen), die den Inhalt umschließen und ihm eine Bedeutung geben. Ein Tag sieht so aus:

HTML 
<!-- Öffnendes Tag, Inhalt, schließendes Tag -->
<p>Das ist ein Absatz.</p>

<!-- Tag mit Attribut -->
<a href="https://example.com">Ein Link</a>

<!-- Selbstschließendes Tag (kein Inhalt) -->
<img src="bild.jpg" alt="Beschreibung">
<br>
<hr>
💡 Merke: HTML beschreibt was etwas ist (Überschrift, Absatz, Link), nicht wie es aussieht. Das Aussehen wird mit CSS gesteuert.

🏗️ Die HTML-Dokumentstruktur

Jede HTML-Seite hat eine grundlegende Struktur, die immer gleich ist. Diese Grundstruktur teilt dem Browser mit, wie er die Seite interpretieren soll.

HTML 
<!DOCTYPE html>
<html lang="de">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <meta name="description" content="Beschreibung der Seite für Suchmaschinen">
  <title>Seitentitel – erscheint im Browser-Tab</title>
  <link rel="stylesheet" href="style.css">
</head>
<body>
  <!-- Hier kommt der sichtbare Inhalt -->
  <h1>Hallo Welt!</h1>
  <p>Das ist meine erste Webseite.</p>

  <script src="script.js"></script>
</body>
</html>

Erklärung der Elemente:

ElementBedeutung
<!DOCTYPE html>Teilt dem Browser mit, dass es sich um HTML5 handelt
<html lang="de">Wurzelelement; lang-Attribut für Sprache (wichtig für Barrierefreiheit)
<head>Enthält Metadaten – nicht sichtbar auf der Seite
<meta charset="UTF-8">Zeichenkodierung – ermöglicht Umlaute (ä, ö, ü)
<meta name="viewport">Wichtig für Responsive Design auf Mobilgeräten
<title>Seitentitel im Browser-Tab und in Suchergebnissen
<body>Enthält den gesamten sichtbaren Inhalt der Seite

📝 Überschriften (h1–h6)

HTML bietet sechs Überschriften-Ebenen. <h1> ist die wichtigste (und größte), <h6> die unwichtigste. Verwende sie hierarchisch – nicht für das Aussehen, sondern für die Struktur.

HTML 
<h1>Hauptüberschrift (nur einmal pro Seite!)</h1>
<h2>Abschnittsüberschrift</h2>
<h3>Unterabschnitt</h3>
<h4>Noch tiefer</h4>
<h5>Sehr selten verwendet</h5>
<h6>Kleinste Überschrift</h6>
⚠️ SEO-Tipp: Verwende <h1> nur einmal pro Seite. Suchmaschinen nutzen Überschriften, um den Inhalt zu verstehen. Überspringe keine Ebenen (z.B. von h2 direkt zu h4).

✍️ Text & Absätze

Für Fließtext gibt es verschiedene HTML-Elemente, je nach Bedeutung des Textes.

HTML 
<!-- Absatz -->
<p>Das ist ein normaler Absatz. Browser fügen automatisch
Abstände zwischen Absätzen ein.</p>

<!-- Fett (semantisch wichtig) -->
<strong>Wichtiger Text</strong>

<!-- Kursiv (semantisch betont) -->
<em>Betonter Text</em>

<!-- Nur visuell fett (keine semantische Bedeutung) -->
<b>Visuell fett</b>

<!-- Nur visuell kursiv -->
<i>Visuell kursiv</i>

<!-- Zeilenumbruch -->
<br>

<!-- Horizontale Linie -->
<hr>

<!-- Kleiner Text -->
<small>Kleingedrucktes, Copyright-Hinweise</small>

<!-- Markierter Text -->
<mark>Hervorgehobener Text</mark>

<!-- Durchgestrichener Text -->
<del>Gelöschter Text</del>

<!-- Eingefügter Text -->
<ins>Eingefügter Text</ins>

<!-- Code im Fließtext -->
<code>console.log('Hallo')</code>

<!-- Zitat -->
<blockquote cite="https://quelle.de">
  Das ist ein längeres Zitat aus einer anderen Quelle.
</blockquote>

🔗 Links (Hyperlinks)

Links sind das Herzstück des Webs. Mit dem <a>-Tag (Anchor) verlinkst du auf andere Seiten, Abschnitte oder Dateien.

HTML 
<!-- Externer Link -->
<a href="https://www.example.com">Externe Website</a>

<!-- In neuem Tab öffnen (+ rel für Sicherheit) -->
<a href="https://www.example.com" target="_blank" rel="noopener noreferrer">
  Neuer Tab
</a>

<!-- Interner Link (andere Seite) -->
<a href="kontakt.html">Zur Kontaktseite</a>

<!-- Anker-Link (Sprung zu einem Abschnitt) -->
<a href="#abschnitt">Zum Abschnitt springen</a>
<section id="abschnitt">...</section>

<!-- E-Mail-Link -->
<a href="mailto:info@example.com">E-Mail senden</a>

<!-- Telefon-Link (für Mobilgeräte) -->
<a href="tel:+491234567890">Anrufen</a>

<!-- Bild als Link -->
<a href="index.html">
  <img src="logo.png" alt="Zur Startseite">
</a>
🔒 Sicherheit: Bei target="_blank" immer rel="noopener noreferrer" hinzufügen. Das verhindert, dass die verlinkte Seite auf deine Seite zugreifen kann (Tabnapping-Angriff).

🖼️ Bilder

Bilder werden mit dem selbstschließenden <img>-Tag eingebunden. Das alt-Attribut ist Pflicht für Barrierefreiheit und SEO.

HTML 
<!-- Einfaches Bild -->
<img src="foto.jpg" alt="Beschreibung des Bildes">

<!-- Mit Breite und Höhe (verhindert Layout-Verschiebungen) -->
<img src="foto.jpg" alt="Beschreibung" width="800" height="600">

<!-- Lazy Loading (Bild erst laden wenn sichtbar) -->
<img src="foto.jpg" alt="Beschreibung" loading="lazy">

<!-- Bild mit Bildunterschrift -->
<figure>
  <img src="diagramm.png" alt="Umsatzdiagramm 2024">
  <figcaption>Abb. 1: Umsatzentwicklung im Jahr 2024</figcaption>
</figure>

<!-- Responsive Bilder mit srcset -->
<img
  src="bild-800.jpg"
  srcset="bild-400.jpg 400w, bild-800.jpg 800w, bild-1200.jpg 1200w"
  sizes="(max-width: 600px) 400px, (max-width: 1000px) 800px, 1200px"
  alt="Responsive Bild"
>
♿ Barrierefreiheit: Das alt-Attribut wird von Screenreadern vorgelesen. Beschreibe das Bild sinnvoll. Bei rein dekorativen Bildern: alt="" (leerer String, nicht weglassen!).

📋 Listen

HTML kennt drei Arten von Listen: ungeordnet, geordnet und Definitionslisten.

HTML 
<!-- Ungeordnete Liste (Aufzählung mit Punkten) -->
<ul>
  <li>HTML</li>
  <li>CSS</li>
  <li>JavaScript</li>
</ul>

<!-- Geordnete Liste (nummeriert) -->
<ol>
  <li>HTML lernen</li>
  <li>CSS lernen</li>
  <li>Erstes Projekt bauen</li>
</ol>

<!-- Geordnete Liste mit Startpunkt -->
<ol start="5">
  <li>Fünfter Punkt</li>
  <li>Sechster Punkt</li>
</ol>

<!-- Verschachtelte Liste -->
<ul>
  <li>Frontend
    <ul>
      <li>HTML</li>
      <li>CSS</li>
    </ul>
  </li>
  <li>Backend</li>
</ul>

<!-- Definitionsliste -->
<dl>
  <dt>HTML</dt>
  <dd>HyperText Markup Language – Struktur von Webseiten</dd>
  <dt>CSS</dt>
  <dd>Cascading Style Sheets – Aussehen von Webseiten</dd>
</dl>

📊 Tabellen

Tabellen sind für tabellarische Daten gedacht – nicht für Layouts! Eine vollständige Tabelle hat Kopfzeile, Körper und optional eine Fußzeile.

HTML 
<table>
  <caption>Monatliche Ausgaben</caption>
  <thead>
    <tr>
      <th scope="col">Monat</th>
      <th scope="col">Einnahmen</th>
      <th scope="col">Ausgaben</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>Januar</td>
      <td>3.500 €</td>
      <td>2.800 €</td>
    </tr>
    <tr>
      <td>Februar</td>
      <td>3.200 €</td>
      <td>2.600 €</td>
    </tr>
  </tbody>
  <tfoot>
    <tr>
      <td>Gesamt</td>
      <td>6.700 €</td>
      <td>5.400 €</td>
    </tr>
  </tfoot>
</table>

<!-- Zellen verbinden -->
<table>
  <tr>
    <td colspan="2">Zwei Spalten breit</td>
  </tr>
  <tr>
    <td rowspan="2">Zwei Zeilen hoch</td>
    <td>Normal</td>
  </tr>
</table>

📝 Formulare

Formulare ermöglichen Benutzereingaben. Sie bestehen aus dem <form>-Container und verschiedenen Eingabeelementen.

HTML 
<form action="/senden" method="POST">

  <!-- Text-Eingabe -->
  <label for="name">Name:</label>
  <input type="text" id="name" name="name"
         placeholder="Dein Name" required>

  <!-- E-Mail -->
  <label for="email">E-Mail:</label>
  <input type="email" id="email" name="email" required>

  <!-- Passwort -->
  <label for="passwort">Passwort:</label>
  <input type="password" id="passwort" name="passwort"
         minlength="8" required>

  <!-- Zahl -->
  <label for="alter">Alter:</label>
  <input type="number" id="alter" name="alter" min="0" max="120">

  <!-- Datum -->
  <label for="datum">Datum:</label>
  <input type="date" id="datum" name="datum">

  <!-- Textarea (mehrzeilig) -->
  <label for="nachricht">Nachricht:</label>
  <textarea id="nachricht" name="nachricht"
            rows="5" placeholder="Deine Nachricht..."></textarea>

  <!-- Dropdown -->
  <label for="land">Land:</label>
  <select id="land" name="land">
    <option value="">Bitte wählen</option>
    <option value="de">Deutschland</option>
    <option value="at">Österreich</option>
    <option value="ch">Schweiz</option>
  </select>

  <!-- Checkboxen -->
  <fieldset>
    <legend>Interessen:</legend>
    <label>
      <input type="checkbox" name="interesse" value="html"> HTML
    </label>
    <label>
      <input type="checkbox" name="interesse" value="css"> CSS
    </label>
  </fieldset>

  <!-- Radio Buttons -->
  <fieldset>
    <legend>Erfahrung:</legend>
    <label>
      <input type="radio" name="erfahrung" value="anfaenger"> Anfänger
    </label>
    <label>
      <input type="radio" name="erfahrung" value="fortgeschritten"> Fortgeschritten
    </label>
  </fieldset>

  <!-- Submit Button -->
  <button type="submit">Absenden</button>
  <button type="reset">Zurücksetzen</button>

</form>
♿ Wichtig: Verknüpfe immer <label> mit dem zugehörigen Input über das for-Attribut (muss mit der id des Inputs übereinstimmen). Das ist wichtig für Barrierefreiheit und verbessert die Klickfläche.

🏷️ Semantisches HTML

Semantische HTML-Elemente beschreiben ihre Bedeutung klar – sowohl für den Browser als auch für Entwickler und Suchmaschinen. Verwende sie statt generischer <div>-Elemente.

HTML 
<!-- Seitenstruktur mit semantischen Elementen -->
<header>
  <nav>
    <ul>
      <li><a href="/">Startseite</a></li>
      <li><a href="/blog">Blog</a></li>
    </ul>
  </nav>
</header>

<main>
  <article>
    <header>
      <h1>Artikel-Titel</h1>
      <time datetime="2024-01-15">15. Januar 2024</time>
    </header>
    <p>Artikel-Inhalt...</p>
    <section>
      <h2>Abschnitt im Artikel</h2>
      <p>Inhalt des Abschnitts...</p>
    </section>
  </article>

  <aside>
    <h2>Verwandte Artikel</h2>
    <!-- Sidebar-Inhalt -->
  </aside>
</main>

<footer>
  <address>
    Kontakt: <a href="mailto:info@example.com">info@example.com</a>
  </address>
  <p><small>© 2024 Meine Website</small></p>
</footer>

Wichtige semantische Elemente:

ElementVerwendung
<header>Kopfbereich der Seite oder eines Abschnitts
<nav>Navigationsbereich mit Links
<main>Hauptinhalt der Seite (nur einmal!)
<article>Eigenständiger Inhalt (Blogpost, News-Artikel)
<section>Thematischer Abschnitt mit Überschrift
<aside>Ergänzender Inhalt (Sidebar, Werbung)
<footer>Fußbereich der Seite oder eines Abschnitts
<figure>Eigenständige Medieninhalte (Bilder, Diagramme)
<figcaption>Bildunterschrift für <figure>
<time>Datum/Uhrzeit (maschinenlesbar)
<address>Kontaktinformationen

💬 Kommentare

HTML-Kommentare sind für Entwickler gedacht und werden vom Browser nicht angezeigt. Sie helfen, den Code zu dokumentieren.

HTML 
<!-- Das ist ein einzeiliger Kommentar -->

<!--
  Das ist ein
  mehrzeiliger Kommentar
-->

<!-- Abschnitt: Navigation -->
<nav>...</nav>

<!-- TODO: Hier noch den Footer ergänzen -->

<!-- Temporär auskommentierter Code:
<div class="banner">
  <p>Werbebanner</p>
</div>
-->
⚠️ Achtung: HTML-Kommentare sind im Quellcode sichtbar! Schreibe keine sensiblen Informationen (Passwörter, API-Keys) in Kommentare.

⭐ Best Practices

✅ Immer valides HTML schreiben

+

Valides HTML wird von allen Browsern korrekt interpretiert. Nutze den W3C Validator um deinen Code zu prüfen.

  • Immer <!DOCTYPE html> am Anfang
  • Tags korrekt schließen
  • Attribute in Anführungszeichen
  • Keine veralteten Tags wie <font> oder <center>

♿ Barrierefreiheit (Accessibility)

+

Barrierefreies HTML ist für alle Nutzer zugänglich – auch für Menschen mit Behinderungen.

  • Immer alt-Attribute bei Bildern
  • lang-Attribut am <html>-Tag
  • Labels für alle Formularfelder
  • Semantische Elemente statt <div>
  • Ausreichende Farbkontraste
  • Tastaturnavigation ermöglichen

🔍 SEO-freundliches HTML

+

Suchmaschinen lesen deinen HTML-Code. Gutes HTML verbessert dein Ranking.

  • Aussagekräftiger <title> (50-60 Zeichen)
  • <meta name="description"> (150-160 Zeichen)
  • Nur ein <h1> pro Seite
  • Semantische Struktur mit <article>, <section> etc.
  • Beschreibende Alt-Texte für Bilder

⚡ Performance

+

Schnelle Webseiten haben bessere Rankings und zufriedenere Nutzer.

  • loading="lazy" für Bilder unterhalb des sichtbaren Bereichs
  • CSS im <head>, JavaScript am Ende des <body>
  • Bilder in modernen Formaten (WebP, AVIF)
  • width und height bei Bildern angeben
  • Externe Ressourcen minimieren
